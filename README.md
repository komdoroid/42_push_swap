# push_swap


## Description（プロジェクトの概要と目的）

push_swap は、2つのスタック（a, b）と限られた11個の操作だけを使って整数列を昇順に並べ替える **操作手順** を生成するプログラムである。
実際にソート結果を出力するのではなく、「この手順を順に実行すれば a がソート済みになる」という操作列（`ra`, `pb` など）を標準出力に書き出す。

| 戦略 | 複雑度クラス | アルゴリズム |
|------|------------|-------------|
| Simple | O(n²) | 挿入ソート（b退避型） |
| Medium | O(n√n) | チャンクソート（√n分割） |
| Complex | O(n log n) | 基数ソート（LSD） |
| Adaptive | 入力依存 | disorder値に応じて上3つを切替（デフォルト） |

戦略はフラグで明示指定でき、無指定の場合は Adaptive が選ばれる。
`--bench` フラグでベンチマーク統計（操作数、disorder、選択戦略）を標準エラー出力に出すこともできる。

また、ボーナスとして checker プログラムを実装している。
checker は push_swap が出力した操作列を実行し、ソートが正しく行えているかを `OK` / `KO` で判定する。

### データ構造の選択

スタックは **双方向循環連結リスト** で実装した。
全11操作（sa/sb/ss/pa/pb/ra/rb/rr/rra/rrb/rrr）を O(1) で実行でき、
コードが操作の意図と1対1で対応するため、保守性と説明のしやすさを重視してこの構造を選んだ。

リングバッファ（配列ベース）も全操作 O(1) で実現できるが、要素移動のたびに
インデックス計算と境界の補正が必要でコードが分散するため採用しなかった。

## Instructions（コンパイル・実行方法）

### ビルド

```sh
make            # push_swap (mandatory) をビルド
make bonus      # checker (bonus) をビルド
make clean      # オブジェクトファイル削除
make fclean     # バイナリも含めて削除
make re         # 再ビルド
```

### push_swap の使い方

```sh
./push_swap 3 1 2 5 4              # 整数を引数で渡す
./push_swap "3 1 2 5 4"            # 文字列形式（1引数に複数値）
```

ソート手順が標準出力に1行ずつ出力される。引数なし、または既にソート済みの場合は何も出力せず正常終了する。

### 戦略フラグ

```sh
./push_swap --simple 3 1 2         # O(n²) を強制
./push_swap --medium 3 1 2         # O(n√n) を強制
./push_swap --complex 3 1 2        # O(n log n) を強制
./push_swap --adaptive 3 1 2       # disorderに応じて自動選択（デフォルト）
```

戦略フラグは互いに排他で、2つ以上指定するとエラー。

### ベンチマークモード

`--bench` を付けると、ソート完了後に統計情報を **標準エラー出力**（fd=2）に出す。
標準出力（操作列）と分離されているため、checker や `wc -l` と併用できる。

```sh
./push_swap --bench 3 1 2                        # 統計をstderrに表示
./push_swap --bench 3 1 2 2>bench.txt | ./checker 3 1 2   # OK/KO + 統計
```

### エラーと特殊な入力

以下は `Error` を標準エラー出力に出力して終了する。

- 整数以外の文字が含まれる
- int の範囲を超える値（INT_MAX 超過、INT_MIN 未満）
- 重複する値
- 不正なフラグ、または戦略フラグの複数指定

以下は何も出力せずに正常終了する。

- 引数なし
- 既にソート済みの入力

### checker の使い方（ボーナス）

```sh
./checker 3 2 1 0                  # 操作列を標準入力から読む
./push_swap 3 2 1 0 | ./checker 3 2 1 0   # パイプで連携
```

引数で受け取ったスタックに対し、標準入力から `\n` 区切りで来る操作列を実行し、
最終的にスタックaがソート済みかつスタックbが空であれば `OK` を、そうでなければ `KO` を標準出力に出す。
不正な引数や未知の操作はエラーとして `Error` を標準エラー出力に出す。

## アルゴリズムの選択理由と複雑度の論証

本セクションでは、各戦略のアルゴリズム選択理由と、それが目標の複雑度クラスを満たすことの論証を述べる。

 なお、本プロジェクトにおける複雑度はすべて **生成される push_swap 操作の数** で測ったものである。

### Simple: 挿入ソート（O(n²)）

最小値を順に b へ退避し、b を降順に保つ。最後に b から a へ pa で戻すと a が昇順に揃う。
双方向回転による最適化
目標値（次に b へ送りたい最小値）が a の中のどこにあるかを判定し、
近い方向（ra か rra）を選んで先頭まで回す。これにより、目標値が下半分にある場合の
回転コストを大幅に削減できる。
複雑度の論証（上界）

各ラウンドの「目標値を先頭に持ってくる」操作: 平均 n/4 回（双方向回転のため、片方向の半分）
ラウンド数: n（全要素を順に b へ送る）
最後の戻し: n 回の pa

総コスト ≈ n × n/4 + n = O(n²)。
オーダーは O(n²) だが、双方向回転により定数倍を約半分に抑えている。
整列に近い入力では各ラウンドの回転距離も小さくなるため、実用上の操作数はさらに少ない。
そのため Adaptive において「要素数が少ない場合、もしくはdisorder が十分小さい入力」に割り当てている。

### Medium: チャンクソート（O(n√n)）

値の範囲を √n 個のチャンクに分割し、a から各チャンクを順に b へ振り分ける。
低い番号のチャンクから順に送ることで、b はチャンクの塊単位で降順に積まれる。
最後に b の最大値を近い方向（rb/rrb）で先頭に持ってきて pa する。

**複雑度の論証（上界）**

チャンク数を k、各チャンクサイズを n/k とすると、

- 振り分けフェーズ: 各チャンクを送るのに最大 n 回の操作（a を1周）。これを k 回繰り返す → O(kn)
- 戻しフェーズ: 各 pa の前に最大 n/k の回転 → O(n²/k)

総コスト f(k) = kn + n²/k。これを k について微分して最小化すると k = √n となり、
このとき総コストは f(√n) = 2n√n = **O(n√n)**。

### Complex: 基数ソート（O(n log n)）

全要素に 0 から n-1 のインデックスを振る（座標圧縮）。
下位ビットから順に、各ビットについて「ビットが0なら b に退避、1なら a に残す」を繰り返し、
b を a に戻す処理を ⌈log₂(n)⌉ 回行う。

**複雑度の論証（上界）**

- 各ラウンドの振り分け: O(n)
- 戻し: O(n)
- ラウンド数: ⌈log₂(n)⌉ = O(log n)

総コスト = O(n) × O(log n) = **O(n log n)**。

インデックス化により扱う値の範囲を 0〜n-1 に圧縮しているため、必要ビット数が log₂(n) に抑えられ、
入力値の絶対的な大きさに依存しないオーダーになる。

### Adaptive: 戦略の動的選択

ソート前に **disorder（乱れ度）** を1度だけ計算し、しきい値で戦略を切り替える。

disorder は「逆転対の数」を「全ペア数 n(n-1)/2」で割った値で、0 から 1 の範囲を取る。
0 に近いほど整列済みに近く、1 に近いほど逆順に近い。

| disorder | 選択戦略 | 理由 |
|----------|---------|------|
| < 0.2 | Simple | 整列に近い入力では逆転対が少なく、挿入ソート的な処理が最も少ない操作数で済む |
| 0.2 〜 0.5 | Medium | 中程度の乱れではチャンクソートが安定した性能を出す |
| ≥ 0.5 | Complex | ランダムに近い入力では、安定したオーダーをもつ基数ソートが有利 |

しきい値（0.2, 0.5）は実測（`--bench` モード）で操作数を測ることで最適値に調整。
